# AI 养老健康评估系统 - 多轮对话问题与数据需求文档

## 文档说明

本文档整理了系统在多轮对话中向用户提出的所有问题，以及需要收集的用户数据字段。用于后续问题设计和优化。

---

## 一、对话流程概述

系统采用**分组追问**的方式，将用户画像分为 **10 个部分**，每部分包含若干相关字段。系统会：

1. 首先显示欢迎语，说明需要收集 10 个部分的信息
2. 逐个分组提问，用户可以用自然语言回答
3. 系统自动提取结构化数据
4. 所有信息收集完毕后，进入确认阶段
5. 用户确认后，生成健康评估报告

**总字段数**：62 个字段

---

## 二、10 个问题分组详细说明

### 第 1 部分：基本信息（G1）

**字段数**：6 个

**提问内容**：
> 先来了解一下基本情况吧😊 老人家今年多大了？是男是女？住在哪个省或城市？是城市还是农村？上过几年学？现在的婚姻状况是怎样（在婚、丧偶还是其他）？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `age` | 年龄 | 整数 | 如：82 |
| `sex` | 性别 | 文本 | 男 / 女 |
| `province` | 省份/地区 | 文本 | 如：北京、河南、广东 |
| `residence` | 居住地类型 | 文本 | 城市 / 农村 |
| `education_years` | 受教育年限 | 整数 | 如：6（小学）、0（未受教育） |
| `marital_status` | 婚姻状况 | 文本 | 已婚 / 丧偶 / 离婚 / 未婚 |

---

### 第 2 部分：健康限制（G2）

**字段数**：1 个

**提问内容**：
> 过去这半年里，有没有因为身体健康的原因，导致日常活动受到限制？比如出门、做家务这类事情——没有限制、有一点限制、还是限制比较严重？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `health_limitation` | 过去6个月是否因健康问题限制了活动 | 文本 | 否 / 是，有些限制 / 是，严重限制 |

**说明**：这是一个重要的健康指标，用于初步判断老人的整体健康状况。

---

### 第 3 部分：日常活动能力 - 基本生活（BADL）（G3）

**字段数**：6 个

**提问内容**：
> 接下来问一下日常生活的基本动作，每项选：不需要帮助、需要部分帮助、还是需要很多帮助👇
> 
> ① 洗澡  ② 穿衣  ③ 上厕所  ④ 室内走动（起床、坐椅子这类）  ⑤ 大小便控制  ⑥ 吃饭
> 
> 您可以直接说，比如'洗澡需要帮助，其他都能自己来'

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `badl_bathing` | 洗澡能力 | 文本 | 不需要帮助 / 部分帮助 / 多方面帮助 |
| `badl_dressing` | 穿衣能力 | 文本 | 不需要帮助 / 部分帮助 / 多方面帮助 |
| `badl_toileting` | 上厕所能力 | 文本 | 不需要帮助 / 部分帮助 / 多方面帮助 |
| `badl_transferring` | 室内活动（起床、坐椅）能力 | 文本 | 不需要帮助 / 部分帮助 / 多方面帮助 |
| `badl_continence` | 大小便控制能力 | 文本 | 不需要帮助 / 部分帮助 / 多方面帮助 |
| `badl_eating` | 吃饭能力 | 文本 | 不需要帮助 / 部分帮助 / 多方面帮助 |

**说明**：BADL（Basic Activities of Daily Living）是评估失能状态的核心指标。

---

### 第 4 部分：日常活动能力 - 工具性活动（IADL）（G4）

**字段数**：8 个

**提问内容**：
> 再问一些稍微复杂一点的日常活动，每项选：能做、有点困难、还是做不了👇
> 
> ① 串门/走亲戚  ② 买东西  ③ 做饭  ④ 洗衣服  ⑤ 走1公里路  ⑥ 提约5斤重的东西  ⑦ 蹲下再站起来  ⑧ 坐公共交通
> 
> 可以逐一说，也可以总结说，比如'走远路和提重物有点困难，其他基本都能做'

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `iadl_visiting` | 串门/走亲访友能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_shopping` | 购物能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_cooking` | 做饭能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_laundry` | 洗衣能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_walking` | 步行1公里能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_carrying` | 提重物（约5kg）能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_crouching` | 蹲下站起能力 | 文本 | 能 / 有点困难 / 不能做 |
| `iadl_transport` | 乘坐公共交通能力 | 文本 | 能 / 有点困难 / 不能做 |

**说明**：IADL（Instrumental Activities of Daily Living）评估更复杂的日常活动能力。

---

### 第 5 部分：慢性病情况（G5）

**字段数**：7 个

**提问内容**：
> 有没有被医生诊断过这些病？每项说有还是没有就好：
> 
> 高血压、糖尿病、心脏病、中风（脑卒中）、白内障、癌症、关节炎/风湿

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `hypertension` | 高血压 | 文本 | 是 / 否 |
| `diabetes` | 糖尿病 | 文本 | 是 / 否 |
| `heart_disease` | 心脏病 | 文本 | 是 / 否 |
| `stroke` | 中风/脑卒中 | 文本 | 是 / 否 |
| `cataract` | 白内障 | 文本 | 是 / 否 |
| `cancer` | 癌症/恶性肿瘤 | 文本 | 是 / 否 |
| `arthritis` | 关节炎/风湿 | 文本 | 是 / 否 |

**说明**：这些是老年人常见的慢性病，对健康风险评估至关重要。

### 第 6 部分：认知功能（G6）

**字段数**：6 个

**提问内容**：
> 现在问一些脑子灵活度的问题，如实回答就好😊
> 
> ① 知道今天是几号吗？② 知道现在是几月份？③ 知道现在是什么季节？④ 知道自己在哪里？
> 
> ⑤ 100减7等于几？再减7呢？再减一次呢？（三道减法题）
> 
> ⑥ 能照着样子画一个简单的图形吗（比如重叠的五边形）？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `cognition_time` | 时间定向（知道今天是几号） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_month` | 月份定向（知道现在是几月份） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_season` | 季节定向（知道现在是什么季节） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_place` | 地点定向（知道自己在哪里） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_calc` | 计算能力（100连续减7，做3次） | 列表 | ['正确','正确','错误'] 等 |
| `cognition_draw` | 画图能力（能否照样画一个简单图形） | 文本 | 正确 / 错误 |

**说明**：认知功能评估用于识别认知障碍和痴呆风险。

---

### 第 7 部分：心理状态（G7）

**字段数**：3 个

**提问内容**：
> 心情方面问几个问题，选从不、很少、有时、还是经常：
> 
> ① 会不会感到情绪低落、提不起劲儿？
> 
> ② 会不会感到紧张焦虑？
> 
> ③ 会不会感到孤独？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `depression` | 是否有抑郁感/情绪低落 | 文本 | 从不 / 很少 / 有时 / 经常 |
| `anxiety` | 是否有焦虑感 | 文本 | 从不 / 很少 / 有时 / 经常 |
| `loneliness` | 是否感到孤独 | 文本 | 从不 / 很少 / 有时 / 经常 |

**说明**：心理健康是老年人整体健康的重要组成部分。

---

### 第 10 部分：社会支持（G10）

**字段数**：6 个

**提问内容**：
> 最后了解一下家庭和支持情况：

> 
> ② 同住有几个人（不算自己）？
> 
> ③ 经济状况自己觉得怎么样？（很好/好/一般/差/很差）

## 四、对话流程示例

### 4.1 完整对话流程

```
系统：您好！我是AI养老健康助手😊

为了给您（或家中老人）生成一份个性化的健康评估报告，我需要了解一些基本情况，
一共分10个部分，每部分几个问题，您可以用自然语言回答，不用太正式。

【第1部分：基本信息】
系统：先来了解一下基本情况吧😊 老人家今年多大了？是男是女？...

用户：老人家今年85岁，女性，住在北京城市，上过6年学，现在丧偶。

系统：（进度：1/10 部分完成，字段完成率 10%）

【第2部分：健康限制】
系统：过去这半年里，有没有因为身体健康的原因...

用户：有些限制，出门和做家务都有点困难。

系统：（进度：2/10 部分完成，字段完成率 11%）

... （继续其他8个部分）

【信息收集完成】
系统：好了，信息都收集完了！（完成度 100%）

📋 基本摘要：年龄：85岁、性别：女、居住地：城市、慢性病：高血压、糖尿病

确认信息无误后，我马上开始分析并生成健康评估报告。
**确认生成报告吗？**（说'确认'或'好的'就开始，说'修改'可以改改）

用户：确认

系统：好的，信息都收集齐了！现在开始帮您分析，大概需要1-2分钟，请稍候⏳

正在进行：① 失能状态判定 → ② 风险预测 → ③ 健康画像 → ④ 知识检索 
→ ⑤ 行动计划 → ⑥ 优先级排序 → ⑦ 报告生成...

【生成报告】
系统：报告已生成完毕✅ 如果有任何疑问，或者想修改某项信息重新生成，直接告诉我就好😊
```

### 4.2 智能提取示例

用户可以一次性提供多个分组的信息：

```
用户：老人家今年85岁，女性，住在北京城市，上过6年学，现在丧偶。
过去6个月因健康问题有些限制了活动。
日常生活能力：洗澡需要部分帮助，穿衣不需要帮助...
慢性病情况：有高血压、糖尿病、白内障、关节炎。
...

系统：（进度：5/10 部分完成，字段完成率 45%）

【第6部分：认知功能】
系统：现在问一些脑子灵活度的问题...
```

系统会自动识别并提取用户提供的所有信息，跳过已填写的部分，只追问缺失的信息。

## 五、数据使用说明

### 5.1 数据验证规则

- **必填字段**：所有 62 个字段都是必填的（系统会持续追问直到填完）
- **数据类型**：
  - 整数：age, education_years, cohabitants, weight, height, income
  - 文本：其他所有字段
  - 列表：cognition_calc（3个元素的列表）

### 5.2 数据用途

收集的数据用于：

1. **失能状态判定**：基于 BADL/IADL 评分
2. **健康风险预测**：短期和中期风险评估
3. **健康画像生成**：功能状态、优势、问题分析
4. **行动计划制定**：个性化照护建议
5. **优先级排序**：紧急程度分级
6. **报告生成**：完整的健康评估报告

## 六、优化建议

### 6.1 当前问题

1. **问题过多**：10 个部分，62 个字段，对话轮次较多
2. **用户体验**：部分问题较专业，用户可能不理解
3. **提取效率**：即使用户一次性提供大量信息，系统提取率仍不够高

### 6.2 优化方向

1. **问题简化**：
   - 合并相似问题
   - 使用更口语化的表达
   - 减少必填字段数量

2. **智能引导**：
   - 根据用户回答动态调整问题
   - 提供示例回答
   - 允许跳过不确定的问题

3. **提取优化**：
   - 提升 LLM 的信息提取能力
   - 增加同义词识别
   - 支持模糊匹配

4. **分组优化**：
   - 考虑将 10 个分组减少到 5-7 个
   - 按重要性排序，优先收集核心信息
   - 非关键信息可选填
