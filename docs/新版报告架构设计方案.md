# 健康评估与照护行动计划 - 新版报告架构设计方案

## 一、新旧版本对比

### 1.1 核心差异

| 维度 | 旧版报告 | 新版报告 |
|------|----------|----------|
| **定位** | 健康评估报告 | 健康评估 + 照护行动计划 |
| **结构** | 6个模块，偏学术 | 5个模块，更实用 |
| **语言** | 相对正式 | 更口语化、接地气 |
| **建议形式** | 分类列举 | 按优先级 + 负责人 + 完成标准 |
| **风险呈现** | 3年/5年概率 | 近期(1-4周) + 中期(1-6月)具体风险 |
| **行动性** | 建议为主 | 可直接执行的行动计划 |

### 1.2 新版报告结构

```
0. 报告说明
   ├── 免责声明
   └── 使用说明

1. 健康报告总结（3句话）
   ├── 整体功能评估
   ├── 核心问题
   └── 干预建议概要

2. 您的健康画像
   ├── 功能状态
   ├── 优势（需保持）
   └── 主要问题（需解决）

3. 风险因素（按时间优先级）
   ├── 近期风险（1-4周）
   └── 中期风险（1-6月）

4. 健康建议（按优先级分级）
   ├── A. 第一优先级（1-3项）
   ├── B. 第二优先级（4-7项）
   └── C. 第三优先级（8-9项）
   
   每项包含：
   - 负责人
   - 怎么做
   - 完成标准

5. 温馨寄语
```

## 三、实现方案

### 3.1 代码结构调整

```python
# 新增类
class ActionPlanAgent(BaseAgent):
    """行动计划生成Agent"""
    def generate_action_plan(self, ...) -> Dict
    
class PriorityAgent(BaseAgent):
    """优先级排序Agent"""
    def rank_actions(self, actions, risks, feasibility) -> List

# 调整类
class RiskAgent(BaseAgent):
    """风险预测Agent - 调整为短期/中期风险"""
    def predict_short_term(self, ...) -> List
    def predict_medium_term(self, ...) -> List
    
class FactorAgent(BaseAgent):
    """因素分析Agent - 新增健康画像"""
    def generate_health_profile(self, ...) -> Dict
    
class ReportAgent(BaseAgent):
    """报告生成Agent - 使用新模板"""
    def generate_report_v2(self, ...) -> str
```


### 3.2 Prompt设计要点

#### RiskAgent 新版Prompt

```
你是老年健康风险预测专家。请按时间维度评估风险，便于家属制定行动计划。

【输出格式】
{
    "short_term_risks": [
        {
            "timeframe": "1-4周",
            "risk": "具体风险名称",
            "trigger": "触发场景（如夜间起床、卫生间）",
            "severity": "高/中/低",
            "preventable": true/false,
            "prevention_key": "预防关键点"
        }
    ],
    "medium_term_risks": [...]
}

【风险识别原则】
1. 优先识别"最要命、最可防"的风险
2. 给出具体触发场景，不要抽象描述
3. 区分可预防 vs 不可预防
4. 用因果链表达中期风险（如：营养↓→肌肉↓→跌倒↑）
```

#### ActionPlanAgent Prompt

```
你是照护行动计划专家。将健康建议转化为可执行的行动计划。

【行动计划要素】
- 负责人：明确谁来做（家属/照护者/老人自己）
- 怎么做：具体步骤，可直接执行
- 完成标准：可验证的结果
- 时间框架：建议完成时间
- 难度/成本/影响：便于优先级排序

【输出原则】
1. 每个行动都要"可落地、可验证、可完成"
2. 避免"建议锻炼"这种模糊表述，要具体到"每天上下午各5-10分钟"
3. 给出"完成标准"而非"做法标准"
4. 考虑农村/城市、独居/同住等实际情况
```

#### PriorityAgent Prompt

```
你是行动优先级排序专家。基于风险紧急度、可行性、成本效益排序。

【排序维度】
1. 风险严重度（会不会要命）
2. 可预防性（做了有没有用）
3. 可行性（能不能做到）
4. 成本效益（投入产出比）

【分级原则】
- A级（第一优先）：3项以内，"最要命、最可防、最好做"
- B级（第二优先）：4-7项，日常维持类
- C级（第三优先）：长期改善类

【输出格式】
为每个行动给出排序理由，如：
"成本低但收益最大" / "风险最高需立即处理" / "长期受益但不紧急"
```

## 四、新旧版本Agent流程对比

### 4.1 旧版流程（6个Agent）

```
输入 → StatusAgent → RiskAgent → FactorAgent 
     → AdviceAgent → ReviewAgent → ReportAgent → 输出
```

### 4.2 新版流程（7个Agent）

```
输入 → StatusAgent → RiskAgent(调整) → FactorAgent(调整)
     → ActionPlanAgent(新增) → PriorityAgent(新增)
     → ReviewAgent → ReportAgent(重构) → 输出
```

### 4.3 数据流转示意

```
UserProfile
    │
    ▼
StatusAgent
    │ status=0/1/2, explanation
    ▼
RiskAgent (调整)
    │ short_term_risks=[], medium_term_risks=[]
    ▼
FactorAgent (调整)
    │ health_profile={functional_status, strengths, problems}
    ▼
ActionPlanAgent (新增)
    │ actions=[{title, responsible, how_to, criteria}]
    ▼
PriorityAgent (新增)
    │ priority_a=[], priority_b=[], priority_c=[]
    ▼
ReviewAgent
    │ quality_check, safety_check
    ▼
ReportAgent (重构)
    │ 使用新模板生成报告
    ▼
最终报告（新格式）
```

## 八、技术实现示例

### 8.1 ActionPlanAgent 核心代码

```python
class ActionPlanAgent(BaseAgent):
    def __init__(self):
        system_prompt = """你是照护行动计划专家...（见上文）"""
        super().__init__("ActionPlanAgent", system_prompt)
    
    def generate(self, status, risks, factors, profile) -> Dict:
        context = f"""
【当前状态】{status['status_name']}
【短期风险】{risks['short_term_risks']}
【中期风险】{risks['medium_term_risks']}
【主要问题】{factors['main_problems']}
【用户画像】年龄{profile.age}，居住{profile.living_arrangement}
        """
        result = self.call_llm(f"请生成行动计划：\n{context}")
        return self.parse_json(result)
```

### 8.2 新版报告生成示例

```python
def generate_report_v2(self, all_results):
    template = REPORT_TEMPLATE_V2
    
    # 填充各部分
    report = template.format(
        disclaimer=self._generate_disclaimer(),
        summary_3_points=self._generate_summary(all_results),
        functional_status=all_results['factors']['functional_status']['level'],
        strengths_list=self._format_list(all_results['factors']['strengths']),
        short_term_risks=self._format_risks(all_results['risks']['short_term']),
        priority_a_actions=self._format_actions(all_results['priority']['priority_a']),
        closing_message=self._generate_closing(all_results)
    )
    
    return report
```

## 九、总结

新版报告架构在保留原有6个Agent核心能力的基础上，通过：
1. **新增2个Agent**（ActionPlanAgent、PriorityAgent）
2. **调整3个Agent**（RiskAgent、FactorAgent、ReportAgent）
3. **重构报告模板**（从评估报告到行动计划）

实现了从"告诉你有什么问题"到"告诉你怎么解决问题"的跨越，大幅提升了报告的实用性和可执行性。
