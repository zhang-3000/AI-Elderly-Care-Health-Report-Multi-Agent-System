# AI 养老健康 Agent 架构设计文档

## 一、项目概述

### 1.1 项目背景
基于中国老年健康长寿调查（CLHLS）1998-2018年数据，构建面向中国老年人的"AI养老健康Agent"，为老年人、家属和基层医生提供：
- 当前功能状态的判定与解释
- 未来失能/健康风险分层与影响因素解析
- 以证据为基础的、可操作的生活方式和照护建议

### 1.2 核心任务
围绕"无失能 → 部分失能 → 严重失能 / 死亡"的转归路径，提供个性化健康评估与干预建议。

### 1.3 数据基础
- **数据来源**：CLHLS 2018年横截面数据
- **样本量**：10,295条记录
- **变量数**：761个原始变量，筛选后约60个核心变量

---

## 二、失能状态定义

### 2.1 评估维度

| 维度 | 项目数 | 具体内容 |
|------|--------|----------|
| **BADL（基本日常生活活动）** | 6项 | 洗澡、穿衣、上厕所、室内活动、大小便控制、吃饭 |
| **IADL（工具性日常生活活动）** | 8项 | 串门、购物、做饭、洗衣、步行1km、提重物、蹲起、公共交通 |

### 2.2 状态分类标准

| 状态 | 名称 | 判定标准 | 委婉表述 |
|------|------|----------|----------|
| **0** | 无失能 | 所有BADL和IADL均能完全自理 | 功能完好 |
| **1** | 部分失能 | 1-3个BADL完全依赖，或1-7个IADL完全不能 | 需要部分协助 |
| **2** | 严重失能 | ≥4个BADL完全依赖，或8个IADL全部不能 | 需要全面照护 |

### 3.2 各Agent职责详解

#### 📊 Stage 1: 状态判定 Agent (StatusAgent)

| 属性 | 说明 |
|------|------|
| **输入** | BADL/IADL 原始数据（6+8=14项） |
| **输出** | 状态标签(0/1/2)、通俗解释、具体受限项目 |
| **核心逻辑** | 基于规则判定：统计BADL依赖数和IADL不能数 |

**输出示例：**
```json
{
    "status": 1,
    "status_name": "部分失能",
    "badl_depend_count": 2,
    "iadl_unable_count": 4,
    "badl_details": ["洗澡", "穿衣"],
    "iadl_details": ["购物", "做饭", "洗衣", "公共交通"],
    "explanation": "日常生活大部分能自理，但洗澡穿衣需要帮助，外出活动受限"
}
```

---

#### 📈 Stage 2: 风险预测 Agent (RiskAgent)

| 属性 | 说明 |
|------|------|
| **输入** | 当前状态 + 风险因素特征（年龄、慢性病、认知、生活方式等） |
| **输出** | 3年/5年转移概率、风险等级(低/中/高)、风险总结 |
| **核心逻辑** | 基于CLHLS文献证据的风险因素权重评估 |

**风险因素权重：**
- 🔴 高风险：年龄≥85、中风病史、认知障碍、多病共存、独居无照护
- 🟡 中风险：年龄75-84、高血压/糖尿病/心脏病、抑郁、缺乏锻炼
- 🟢 保护因素：规律锻炼、良好认知、家庭支持、完善医保

**输出示例：**
```json
{
    "current_status": 1,
    "risk_3year": {
        "maintain": "中",
        "improve": "低",
        "worsen": "中",
        "adverse": "中"
    },
    "overall_risk_level": "中",
    "risk_summary": "高龄和缺乏锻炼增加风险，但家庭支持是重要保护因素"
}
```

---

#### 🔬 Stage 3: 因素分析 Agent (FactorAgent)

| 属性 | 说明 |
|------|------|
| **输入** | 全部特征 + 状态判定结果 + 风险预测结果 |
| **输出** | 可改变因素、不可改变因素、因素贡献度、文献支撑 |
| **核心逻辑** | 区分可干预与不可干预因素，识别改善空间 |

**因素分类：**

| 类型 | 因素示例 | 说明 |
|------|----------|------|
| **不可改变** | 年龄、性别、既往中风、教育程度 | 需认识并接纳 |
| **可改变** | 体力活动、慢性病管理、认知训练、社会参与、营养状态、情绪管理 | 行动重点 |

---

#### 💡 Stage 4: 建议生成 Agent (AdviceAgent)

| 属性 | 说明 |
|------|------|
| **输入** | 状态 + 风险 + 因素分析 + 用户画像 |
| **输出** | 生活方式建议、功能训练、认知社交、照护资源、就医建议 |
| **核心原则** | 具体可操作、循序渐进、安全第一、积极正向 |

**建议类别：**

1. **生活方式**：运动频次/时长、饮食结构、睡眠、戒烟限酒
2. **功能训练**：力量训练、平衡训练、防跌倒措施
3. **认知社交**：社交活动、兴趣爱好、记忆训练
4. **照护资源**：长护险、居家服务、康复服务
5. **就医建议**：体检频次、专科随访、紧急情况识别

---

#### ✅ Stage 5: 反思校验 Agent (ReviewAgent)

| 属性 | 说明 |
|------|------|
| **输入** | 所有前序Agent的输出结果 |
| **输出** | 一致性检查、安全性检查、敏感词替换、质量评估 |
| **核心职责** | 质量控制与安全把关 |

**检查要点：**
- ✓ 一致性：状态判定与风险预测是否矛盾
- ✓ 合理性：建议是否与因素分析匹配
- ✓ 安全性：是否有需要紧急就医的情况
- ✓ 敏感词：替换"失能""死亡"等词汇
- ✓ 完整性：是否覆盖所有必要内容

**敏感词替换规则：**

| 原词 | 替换为 |
|------|--------|
| 失能 | 功能受限 / 需要帮助 |
| 死亡 | 不良健康结局 / 健康风险 |
| 严重失能 | 需要较多照护支持 |

---

#### 📝 Stage 6: 报告生成 Agent (ReportAgent)

| 属性 | 说明 |
|------|------|
| **输入** | 所有评估结果 + 用户类型 |
| **输出** | 结构化Markdown报告 |
| **核心职责** | 整合输出、调整语言风格 |

**报告结构：**
1. 简要总结（2-3句话）
2. 当前状态判定与解释
3. 未来风险分层
4. 关键影响因素说明
5. 个体化建议
6. 就医与安全提醒

**语言风格适配：**
| 用户类型 | 语言风格 |
|----------|----------|
| 老年人/家属 | 简洁温和、避免术语、强调积极面 |
| 医生/专业人员 | 可用专业术语、提供具体数据 |

## 六、实验验证

### 6.1 实验设计

| 项目 | 说明 |
|------|------|
| 样本量 | 50条（分层抽样） |
| 标签分布 | 无失能17条、部分失能17条、严重失能16条 |

### 6.2 评估指标

- **准确率 (Accuracy)**：整体预测正确率
- **分类别准确率**：各失能状态的识别准确率
- **报告质量**：人工评估合理性

| 案例2 | 95岁 | 男 | 部分失能 | 部分失能 ✓ | 高龄、缺乏锻炼、认知下降 |
| 案例3 | 100岁 | 女 | 严重失能 | 严重失能 ✓ | 高龄、肥胖、关节炎 |

## 七、报告输出示例

### 7.1 报告结构

```markdown
# 健康评估报告

## 1. 简要总结
（2-3句话概括当前状态和风险趋势）

## 2. 当前状态判定与解释
（明确状态分类 + 通俗解释判定依据）

## 3. 未来风险分层
（3年/5年风险预测 + 强调是相对概率）

## 4. 关键影响因素说明
- 不可改变因素：年龄、既往疾病...
- 可改变因素：活动、饮食、情绪...

## 5. 个体化建议
- 优先行动（3项）
- 生活方式建议
- 功能训练建议
- 照护资源推荐

## 6. 就医与安全提醒
（紧急情况识别 + 定期随访建议）
```

### 7.2 语言风格对比

**面向老年人/家属：**
> 您目前的身体功能状态需要家人较多的协助来完成部分日常活动，这是高龄长者中常见的情况。通过一些安全、温和的调整，可以帮助您维持现有的活动能力。

**面向医生/专业人员：**

> 患者当前处于部分失能状态（BADL依赖2项，IADL不能4项），基于多状态Markov模型预测，3年内功能恶化概率约40%，建议增加随访频次并进行综合老年评估（CGA）。
