面向基层连续健康管理的专业工作入口，用于快速掌握老年人整体状况、识别重点风险、支持分层管理、生成随访与干预建议，并服务于签约家庭医生团队的日常管理工作。（待定）

核心不应是“陪聊式问答”，而应是决策支持入口
可以通过这个端口快速了解老年人的基本情况，如：
这位老人现在风险高不高
主要问题在哪里
哪些问题优先处理
是否需要上门、复诊、转诊
接下来怎么管理
预览+补充

主要使用场景
场景1：签约居民的首次健康评估
适用于：
新纳入家庭医生签约服务的老人
新建老年健康档案
社区年度体检后补充评估
慢病建档时同步评估功能状态和照护需求

场景2：日常随访和慢病管理
适用于：
高血压/糖尿病常规随访
老年综合评估复查
家庭医生签约后的周期管理
社区卫生服务中心日常电话或门诊随访

场景3：发现异常后的风险复核
适用于：
老人近期跌倒
家属反映记忆下降
近期住院后回社区
近期活动能力明显下降
情绪异常、睡眠异常
出现用药依从性问题

场景4：居家管理与上门服务
适用于：
失能、半失能老人
高龄独居老人
出院后回家老人
有长期照护需求的老人


模块预览示例：
模块1：患者总览
最关键的信息摘要：
姓名、年龄、性别
签约状态
居住情况（独居/与家属同住）
慢病概况
当前风险等级
功能状态等级

最核心的页面之一。
“风险标签 + 优先级”展示：
跌倒风险：低/中/高
失能风险：低/中/高
认知问题风险：低/中/高
抑郁/情绪风险：低/中/高
慢病管理风险：低/中/高
用药风险：低/中/高
社会支持不足风险：低/中/高
近1个月是否有变化
最近一次评估时间
本次主要问题提示

高风险的主要原因是什么。
例如：
近期起身困难，近3个月有跌倒史，夜间如厕频繁
家属反馈记忆下降明显，重复提问增多
独居且近期活动能力下降，缺少稳定照护

模块3：信息来源与一致性判断
这点在医生端非常有用，也具有我们 agent 的特色。
信息来源：
老人本人
家属
医生
历史档案

所以医生端应明确标出：
哪些信息来自老人自述
哪些来自家属观察
哪些是医生确认
哪些信息彼此不一致

例如：老人自述“基本能独立洗澡”，但家属反馈“洗澡需要协助”，建议进一步核实。

模块4：综合评估详情
咱们生成的报告：
例如：
4.1 基本情况
年龄、婚姻、教育、居住、经济与医保情况等。
4.2 功能状态
BADL、IADL、活动能力、近阶段变化。
4.3 慢病与健康状况
高血压、糖尿病、心脑血管病、关节问题、视听障碍等。
4.4 认知与心理
时间定向、记忆、计算、情绪、孤独感等。
4.5 生活方式
吸烟、饮酒、运动、睡眠。
4.6 家庭照护与支持
同住情况、主要照护者、家庭支持、照护难点。

医生可以点开看细节，但默认还是摘要模式。（待定）

模块5：本次随访/看病的发现
本次建议动作
医生端一定要支持记录“我做了什么”。
例如：
1个月内复评 BADL/IADL
建议家属加强夜间照护
建议排查居家跌倒隐患
建议强化药物管理提醒

必要时转上级医院进一步认知评估

模块6：记录与管理
本次随访方式：门诊/电话/上门
给出的建议
是否联系家属
是否安排复诊/复评
是否转诊
下次随访时间
为下一次做好准备
它不是一个“只会分析”的工具，而是能进入基层管理闭环。
